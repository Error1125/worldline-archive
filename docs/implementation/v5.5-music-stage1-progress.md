# v5.5 Music Archive — Stage 1

Stage 1 establishes a manually curated music archive. `appleMusic` stores only public metadata and optional preview URLs; no Apple API, token, OAuth flow, or Worker deployment is involved.

The mapper preserves legacy `appleMusicUrl`, `cover`, `comment`, and `mood` while preferring the new object fields. The playlist is initialized from public music content and persisted by `trackId`; a real `HTMLAudioElement` is created only after an explicit preview click. Missing previews do not simulate playback.

The music page is a featured record and vinyl shelf. Motion owns the list wrapper; the card owns disc hover/rotation and honors reduced motion. Admin and Worker source support the new fields, but **Worker deployment pending**. Stage 2 may add an explicitly authorized Apple catalog integration.
