# v5.3.1 Motion Manual QA

Start: `npx.cmd pnpm@9.15.9 dev -- --host 127.0.0.1`.

Test each URL at 1440x900 and 375x812: `/`, `/moments`, `/timeline`, `/anime`, `/projects`, `/photos`, `/music`, `/posts`, `/bugs`, `/search`.

- Verify search, filter, sort, empty result and rapid repeated input where provided.
- Navigate `/` -> `/moments` -> `/anime` -> `/projects` -> `/photos` -> `/music` -> `/posts` -> `/bugs` -> `/search` -> `/timeline`; then browser back/forward. Header, background, theme and global player must remain stable.
- At `/timeline`, switch Timeline -> Observation Index -> Timeline and rapidly alternate both tabs. Check 1440x900 and 375x812: only the content panel transitions, controls stay still, no panel remains clickable after cancellation, no horizontal overflow, and reduced-motion switches with direct/short opacity behavior.
- At `/timeline`, confirm there is exactly one search/filter/sort/count toolbar. Enter a query, select a type and sort order, then switch both directions: the query, selected chip, order and result count must remain unchanged. Clear the query and verify the common empty state in both views.
- Open the worldline sort control and verify it is a themed listbox (not a native system select). Check ArrowUp/ArrowDown navigation, Enter/Space selection, Escape close with focus returned to the trigger, click-outside close and no clipping at 1440x900 or 375x812.
- Visit `/timeline?view=index`, then use tabs and browser Back/Forward. URL, selected tab and visible view must remain synchronized. Rapid Timeline -> Observation -> Timeline -> Observation switching must settle on Observation with only one interactive panel.
- In Photos, open/close Lightbox, previous/next, Escape and focus return.
- Emulate `prefers-reduced-motion: reduce`: content remains visible and interactive; route/list/lightbox use only short opacity or direct transitions.
- In Admin, verify the existing native confirmation workflow: confirm is shown, Cancel performs no dangerous action, OK runs once, prompt input/cancel work, and browser Enter/Escape semantics remain normal. Do not expect custom dialog, drawer or toast animation; these are deferred to a dedicated Admin Interaction System.

Record any failure with URL, viewport, action, screenshot and browser-console message. The controlled-browser session was unavailable during implementation, so this checklist is required before declaring full browser acceptance.
