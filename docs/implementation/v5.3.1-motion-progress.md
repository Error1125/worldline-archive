# v5.3.1 Motion Architecture Progress

## Current baseline

- Branch: `main`, currently based on `f248785 fix(v5.3): prevent filter cards from flying in`.
- The working tree contains an uncommitted Moments Motion React Island. It is the protected visual-quality baseline for this iteration and must not be reverted to the previous `data-filter-root` / WAAPI implementation.
- Existing untracked handoff and historical documents are user-owned and intentionally excluded.

## Moments quality baseline

- `LayoutGroup` + `AnimatePresence mode="popLayout"` + stable `moment.id` keys.
- Layout spring: stiffness 165, damping 24, mass 0.85.
- Enter: opacity 0→1, y 20→0, scale .985→1; exit: opacity 1→0, y 0→8, scale 1→.96.
- The filter toolbar remains outside the animated list. The list outer wrapper owns layout; each card outer wrapper owns enter/exit/layout transform.

## Completed pages

- Moments: protected React Motion Island baseline.
- Timeline: `TimelineMotion` replaces the old filter root; line growth, ordered nodes, node-local card enter and true chronological regrouping are Motion-owned.
- Observation Index: `ObservationMotion` replaces the old filter root; its search, filters and sort use pop-layout and scan/focus opacity + low blur semantics.
- Anime: `AnimeFeed` is a React Motion Island with status/tag filters, 100ms debounced search, chronological sort, stable keys and pop-layout exit. Browser acceptance remains pending because the browser-control session failed before interaction could be run.
- Projects: `ProjectsFeed` is a React Motion Island with status/tech filters, 100ms debounced search, chronological sort, stable keys and pop-layout exit. Browser acceptance remains pending for the same reason.

## Motion Foundation

- `src/lib/motion/presets.ts`: motion tokens, scene enter/exit presets, shared layout spring and reduced-motion variants.
- `src/components/motion/MotionList.tsx`: `LayoutGroup` + `AnimatePresence mode="popLayout"` list shell.
- `src/components/motion/MotionItem.tsx`: the only transform owner for a Motion list item's outer layer.
- `src/components/motion/CollapseMotion.tsx` and `SceneMotion.tsx`: state and page-local primitives.

- Remaining legacy-filter list wrappers now use opacity-only first-view animation; CSS transforms are limited to named card internals so legacy WAAPI positioning cannot compete with a scene keyframe on the same DOM node.
