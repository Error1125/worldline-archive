## Interaction System

- Persistent Admin Shell: `AdminApp` mounts one router-backed shell for authenticated admin routes, keeping navigation and topbar stable while swapping the content outlet.
- Router: `AdminRouterProvider` parses deep links, pushes internal navigation, handles `popstate`, and supports async navigation guards.
- Sidebar / Drawer: desktop uses persistent sidebar navigation; mobile uses a focus-trapped drawer with overlay and Escape close behavior.
- Content Outlet: `AdminContentOutlet` animates route changes and keeps the shell outside the replaced content area.
- Button System: `Btn` provides shared button variants, loading state, icon-style buttons, and disabled state.
- Listbox: themed listbox replaces old tab/select-style admin controls where the v5.4 UI needs custom keyboard navigation and visual consistency.
- Toast: `ToastProvider` renders timed success/error/warning/info feedback and clears timers.
- Dialog / Prompt: `DialogProvider` replaces native confirm-style destructive flows with promise-based modal confirmation. Current permanent delete confirmation is implemented as a danger confirm requiring slug input inside the dialog copy and action flow.
- Focus Trap: dialog and drawer layers trap focus, restore body scroll, and release on close.
- Loading / Skeleton: `Skeleton`, `ContentSkeleton`, `EmptyState`, and `ErrorState` cover route and data loading states.

## Screens

- Content Manager: supports type/status/search filtering, editor dirty tracking, save status, reversible hide/restore confirmation, permanent delete confirmation, and save toast feedback.
- Settings: site/profile/worldline/bangumi sections route internally under the persistent shell and refresh directly from their deep links.
- Publish: publish index and form routes remain compatible with Astro `screen` / `publishType` props and validate required fields before submit.
- Unsaved Guard: content editor and publish/settings forms register route guards and `beforeunload` prevention while dirty.
